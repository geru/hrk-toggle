Responsive Navbar, menus, page elements, and hierchical trees without using Javascript

Most responsive HTML elements, and especially navigation bars and menus employ Javascript and a Jquery or Jquery-like Javascript library to figure out when the website needs to do a layout change. Jquery-like libraries are fine and good, but they are also constantly changing, so the disadvantages of this approach are:

  • Dependency on a particular version of the Jquery-like library: When that library changes or upgrades, then behavior will change. When this happens, you may have to debug Javascript as well as CSS and HTML to get to the bottom of a new problem. You may also run into the need to keep the older Javascript library for one purpose while simultaneously loading a newer version of the same library for another purpose on the same webpage... and then, things get unnecessarily complicated.
  • Browser workload: The browser has to process the Javascript library and to do a bunch of other work before the page can load properly.

So, using Jquery-like libraries for responsive behaviors wastes bandwidth loading extra Javascript libraries, and then makes browsers process piles of extra code. And, the website developer gets a new problem to solve (debugging Javascript), which may not be their bailiwick. In other words, the page loads slower and is less reliable in the long term.

Using the checkbox-hack that has now become supported by almost every web browser, you can accomplish many interesting things, including responsive navigation bars, hierarchical tree navigation, and responsive page elements, simply by using the hrk-toggle CSS library and particularly, structured HTML. The only downside is that the HTML is more complicated than simple HTML list tags.

This page is an example page that shows how the extremely tiny hrk-toggle CSS library can be used to create:

  • a responsive navigation bar at the top,
  • responsive sidebars left and right,
  • a navigable hierarchical tree inside the left sidebar,
  • a responsive footer...
all without using a single speck of Javascript.

Right sidebar information is responsive, and toggled on and off with the ellipsis toggle format
