Responsive Navbar, menus, page elements, and hierchical trees without using Javascript

Most responsive HTML elements, and especially navigation bars and menus employ Javascript and a Jquery or Jquery-like Javascript library to figure out when the website needs to do a layout change. Javascript is fine and good, but it is constantly changing, so the disadvantages of this approach are:

  • Dependency on a particular version of the Jquery-like library, and when that library changes or upgrades, then behavior may change. When this happens, you will have to debug Javascript as well as CSS and HTML to get to the bottom of the problem. You may also run into the need to keep the older Javascript library for front-end while simultaneously loading a newer version of the same library... and then, things start to get unnecessarily complicated.
  • The Javascript library has to do a bunch of work before the page can load properly.

So, you are causing the user to waste bandwidth loading extra Javascript libraries, and then making their browser process piles of code. And, you are giving your developer a new problem to solve (debugging Javascript), which may not be their bailiwick. And, the page loads slower, uses more bandwidth, and is less reliable in the long term.

Using the checkbox-hack that has now become supported by almost every web browser, you can accomplish many interesting things, including responsive navigation bars, hierarchical tree navigation, and responsive page elements, simply by using the hrk-toggle CSS library and particularly, structured HTML.

This page is an example page that shows how the tiny hrk-toggle CSS library can be used to create:

  • a responsive navigation bar at the top,
  • responsive sidebars left and right,
  • a navigable hierarchical tree inside the left sidebar,
  • a responsive footer...
all without using a single speck of Javascript.
